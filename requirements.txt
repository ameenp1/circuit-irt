#   python3.12 -m venv .venv
#   .venv/bin/pip install --upgrade pip
#   .venv/bin/pip install --only-binary=:all: -r requirements.txt
#
# Platform constraints that drove these versions (Intel macOS):
#   * torch is capped at 2.2.2 — the last PyTorch release with x86_64 macOS
#     wheels. torch 2.2.2 was built against NumPy 1.x.
#   * numpy is therefore pinned <2 (1.26.4); numpy 2.x breaks torch 2.2.2's
#     array interop ("Failed to initialize NumPy").
#   * transformers is held at 4.49.0 — transformers 5.x requires torch>=2.4
#     and silently disables the PyTorch backend on torch 2.2.2.
#   * numpy<2 pulls pymc down to the 5.x line (pymc 6.x's arviz requires
#     numpy>=2); pymc 5.25.1 + pytensor 2.31.7 work with numpy 1.26.
#   * numba/llvmlite pinned to wheeled versions (0.62.1/0.45.1); llvmlite
#     0.47 has no x86_64 macOS wheel and fails to build from source.
#
# On Apple Silicon / Linux you can relax most of these (newer torch + numpy 2.x).
